### 2. Toggle removed, autoservice always-on (`f1ebb2c`)

User-facing переключатель «Системные данные (экспериментально)» / «System data (experimental)» (`autoservice_enabled`) **удалён полностью**. Нативный autoservice read+write путь теперь **ALWAYS-ON**:

- SoH/lifetime, автоматическое логирование зарядок, ABRP engine power, live-детект зарядного пистолета работают **по умолчанию**, без переключателя в Settings и без шага в first-run wizard.
- Fail-soft на машинах без autoservice (старая прошивка, не-DiLink) сохранён через `isAvailable()` / sentinel guards.
- `detectOfflineCharge` делегирует `AutoserviceChargingDetector`.

Удалены `autoservice_enabled` строки из `values/strings.xml` и `values-en/strings.xml`, поля из `SettingsRepository` / `SettingsViewModel`, UI из `SettingsScreen`. Зачищены условные ветки в `BatteryStateRepository`, `ChargesScreen/ViewModel`, `DashboardScreen/ViewModel`, `BatteryHealthDialog`, `TrackingService`.

### 3. Native trips on non-energydata cars (closed)

Машины **без** BYD energydata SQLite (старая прошивка, не-Leopard3 типа Song/Yuan) теперь пишут поездки нативно через `TripRecorder` (source=NATIVE_POLLING). Старое ограничение «поездки недоступны / список пуст на не-Leopard3» **закрыто**. Расход из SOC delta грубее, чем energydata BMS, но список поездок больше не пустой. На Leopard 3 (energydata есть) `TripRecorder` пассивен, поездками занимается `HistoryImporter` как раньше.

## What remains before release (in order)

### 1. Rebuild + sign

APK на DiLink предшествует этим 4 коммитам, поэтому обязателен пересбор:

```bash
export JAVA_HOME=/opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk/Contents/Home
export ANDROID_HOME=$HOME/Library/Android/sdk
./gradlew assembleRelease
$ANDROID_HOME/build-tools/34.0.0/apksigner sign \
  --ks $HOME/.android/keystores/bydmate-release.jks --ks-key-alias bydmate \
  --ks-pass pass:bydmate123 --key-pass pass:bydmate123 \
  --out BYDMate-v2.9.0.apk app/build/outputs/apk/release/BYDMate-v2.9.0.apk
```

Версия НЕ бампается (одна версия = один набор проверенных изменений). Тот же package + тот же keystore сохраняют апгрейд in-place. Деплой: `adb push` в `/sdcard/Download`, никогда `adb install`. Andy ставит вручную через файловый менеджер.
