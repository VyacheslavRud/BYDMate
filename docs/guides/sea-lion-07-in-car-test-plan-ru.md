# План проверки BYDMate Dev в BYD Sea Lion 07

Цель — за один подход проверить новый read-only контракт приборки и дополнительные scalar-поля
лобового HUD, не изменяя рабочую Waze-навигацию.

## Подготовка

1. Соберите или используйте `BYDMate-v3.6.34-dev.apk` (`versionCode=402`). Новая версия важна:
   приложение по ней распознает старый helper-daemon и перезапустит его с актуальным C09.
2. Установите APK рядом с оригинальным BYDMate:

```bash
adb devices -l
adb -s SERIAL_МАШИНЫ install -r app/build/outputs/apk/debug/BYDMate-v3.6.34-dev.apk
```

3. Откройте `BYDMate Dev` и подождите 5–10 секунд. В Центре диагностики фоновый сервис и helper
   должны быть доступны.
4. Все лабораторные проверки выполняйте только когда машина стоит, включена передача P, скорость
   равна 0 км/ч и в Waze нет активного маршрута.
5. Не запускайте `HUD Lab` и `Instrument Cluster Lab` одновременно.

## Лобовой HUD: расширенные поля

Откройте `Центр диагностики` -> `HUD Lab` -> `Расширение HUD`. Подтвердите P непосредственно перед
запуском. Каждый сценарий сам очистит кадр приблизительно через 8 секунд.

Рекомендуемый порядок:

1. `HX01 SPEED · f11=50, f6=1` — запишите, появилось ли отдельное число/круг скорости вместе со
   стрелкой. Не принимайте число дистанции слева за speed limit.
2. `HX05 RENDER CLASS · f6=6 + f11=50` — сравните с HX01: изменился ли круг, число или расположение.
3. `HX02 ETA · f26=12:34` — ищите именно `12:34`, а не системное время машины.
4. `HX03 PROGRESS · f33=0.5` — ищите полосу или иной индикатор половины маршрута.
5. `HX04 FULL SCALAR` — проверьте совместное отображение улицы `HUD LAB FULL`, стрелки, `50`,
   `12:34` и progress.

После каждого теста выберите в приложении максимально точный визуальный результат. Если кадр
мелькнул и исчез, отмечайте `Появилось и пропало`, а не `Ничего`.

## Приборка за рулём

Откройте `Центр диагностики` -> `Instrument Cluster Lab`.

1. Сначала выполните `C09 IAutoContainer contract snapshot`. На приборке ничего появиться не
   должно: это read-only чтение descriptor и `getProjectionDisplayInfo`. Успех проверяется в
   экспорте по `descriptorCaptured=true`, `projectionInfoCaptured=true` и `mutation=NONE`.
2. Затем выполните `C08 Manual factory Navi watch`. Сначала дождитесь, пока приложение закончит
   базовый снимок и покажет шаг `switch_factory_navi_now`. Только после этого кнопками руля
   переключите область приборки на штатную страницу/карточку навигации BYD. Маршрут и интернет
   штатной карте не нужны. В течение 20 секунд отметьте, появилась ли пустая китайская карта,
   изменился ли экран или ничего не произошло.
3. Последним при необходимости выполните `C07 Sea Lion container transport`. Он отправляет только
   прежнюю whitelist-последовательность 16/18/0, собирает три системных снимка и обязательно
   завершает cleanup. Нормально, если на время появится штатная пустая китайская оболочка карты.

`C03–C06` для этой проверки не нужны. Они относятся к Android display projection, а собранный
системный стек показывает, что физическая приборка Sea Lion использует native Qt/Fission path.

## Контроль рабочей Waze-навигации

После завершения лабораторий нажмите `Очистить HUD`, вернитесь в Waze и только тогда задайте
короткий маршрут. Проверьте, что рабочий лобовой HUD по-прежнему показывает дистанцию, улицу и
правильную стрелку без мигания. Новые HX-сценарии не включаются в production автоматически.

## Экспорт

В HUD Lab нажмите `Экспортировать`. Общий диагностический файл включает журнал лобового HUD и
Instrument Cluster Lab. Перед отправкой убедитесь, что в нём есть записи `HX01–HX05`, `C09` и,
если они выполнялись, `C08/C07`.

Если C09 не содержит нового раздела `auto_container_projection_info`, полностью закройте и снова
откройте `BYDMate Dev`. APK с `versionCode=402` должен автоматически заменить старый helper-daemon.
