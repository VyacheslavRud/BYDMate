# План проверки BYDMate Dev в BYD Sea Lion 07

Цель — за один короткий подход снять точный read-only inventory поверхностей приборки, не изменяя
рабочую Waze-навигацию и не повторяя уже бесполезные HUD-тесты.

## Подготовка

1. Соберите или используйте `BYDMate-v3.6.35-dev.apk` (`versionCode=403`). Новая версия важна:
   приложение по ней распознает старый helper-daemon и перезапустит его с актуальным C09.
2. Установите APK рядом с оригинальным BYDMate:

```bash
adb devices -l
adb -s SERIAL_МАШИНЫ install -r app/build/outputs/apk/debug/BYDMate-v3.6.35-dev.apk
```

3. Откройте `BYDMate Dev` и подождите 5–10 секунд. В Центре диагностики фоновый сервис и helper
   должны быть доступны.
4. Все лабораторные проверки выполняйте только когда машина стоит, включена передача P, скорость
   равна 0 км/ч и в Waze нет активного маршрута.
5. Не запускайте `HUD Lab` и `Instrument Cluster Lab` одновременно.

## Лобовой HUD

`HX01–HX05` больше запускать не нужно: предыдущая проверка уже доказала, что эти scalar-поля на
вашей прошивке не дают speed/ETA/progress и только повторяют известные стрелки. Набор удалён из UI.
Рабочий production-путь лобового HUD этой сборкой не изменяется.

## Приборка за рулём

Откройте `Центр диагностики` -> `Instrument Cluster Lab`.

1. Выполните `C09 Fission projection contract snapshot`. На приборке ничего появиться не должно:
   тест только читает `AutoContainerNative`, Java bridge `auto_container` и inventory
   `FissionHostSvc`. В экспорте нужны секции `auto_container_projection_info`,
   `auto_container_attempt_1/2` и `fission_projection_inventory`; итоговая строка содержит
   `projectionInfoSource`, `fissionStatus` и список `fissionDisplays`.
2. Только если `C09` найдёт неожиданную дополнительную поверхность, затем имеет смысл выполнить
   `C08 Manual factory Navi watch`. Сначала дождитесь, пока приложение закончит
   базовый снимок и покажет шаг `switch_factory_navi_now`. Только после этого кнопками руля
   переключите область приборки на штатную страницу/карточку навигации BYD. Маршрут и интернет
   штатной карте не нужны. В течение 20 секунд отметьте, появилась ли пустая китайская карта,
   изменился ли экран или ничего не произошло.
3. `C07 Sea Lion container transport` сейчас не нужен. Он отправляет только
   прежнюю whitelist-последовательность 16/18/0, собирает три системных снимка и обязательно
   завершает cleanup. Нормально, если на время появится штатная пустая китайская оболочка карты.

`C03–C06` для этой проверки не нужны. Они относятся к Android display projection, а собранный
системный стек показывает, что физическая приборка Sea Lion использует native Qt/Fission path.

## Контроль рабочей Waze-навигации

После завершения лабораторий нажмите `Очистить HUD`, вернитесь в Waze и только тогда задайте
короткий маршрут. Проверьте, что рабочий лобовой HUD по-прежнему показывает дистанцию, улицу и
правильную стрелку без мигания. `C09` не отправляет HUD-кадры и не меняет production-навигацию.

## Экспорт

В HUD Lab нажмите `Экспортировать`. Общий диагностический файл включает Instrument Cluster Lab.
Перед отправкой убедитесь, что в нём есть `C09`, `fission_projection_inventory` и, если он
выполнялся, `C08`.

Если C09 не содержит раздела `fission_projection_inventory`, полностью закройте и снова откройте
`BYDMate Dev`. APK с `versionCode=403` должен автоматически заменить старый helper-daemon.
